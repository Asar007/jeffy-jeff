Critical Issues — Fix This Week

CRITICAL 1Four of five navigation links return 404

/about, /projects, /customization, and /contact all return HTTP 404. Every procurement persona abandons at the first click. The hero "Start a Conversation" CTA and all five footer social icons route to /contact — every conversion path is dead.

Fix: Deploy the inner pages, or as a same-day stopgap, redirect those routes to homepage anchors (/#about, /#projects, etc.). Replace the contact CTA with a working mailto:md@madrasswastic.com link immediately.

CRITICAL 2Confirm custom domain is primary (was on Vercel slug)

The baseline canonical was rocket-ship-plum.vercel.app — zero brand signal, zero backlink equity. AI engines cluster authority around named brand domains.

Fix: In Vercel → Project → Domains, confirm madrasswastic.in is set primary and the slug auto-301s. Update every canonical, og:url, and schema @id to https://madrasswastic.in.

CRITICAL 3/robots.txt and /sitemap.xml both return 404

AI crawlers (GPTBot, ClaudeBot, PerplexityBot, OAI-SearchBot) have no policy signal — cautious operators default to restricted crawling. Google has no clear page inventory.

Fix: Publish both at the root.

CRITICAL 5No RFQ / vendor pre-qualification path

The only CTA leads to a 404. No quote form, no downloadable capability brochure, no ISO certificate PDF, no project portfolio. B2B procurement teams cannot self-qualify the vendor and will not email blindly.

Fix: Build /rfq with structured fields (Project type [Structural Fab / Plant Erection / Piping / Turnkey], Tonnage estimate, Location, Timeline, Message). Publish the ISO 9001:2015 certificate as a downloadable PDF. Add a /capability-brief.pdf with crane tonnage, weld capabilities, equipment list.

High-Priority Issues

HIGH 6H1 contains zero target keywords

Current H1: "Engineering Precision for Modern Infrastructure" — brand copy, not a search signal.

Fix: H1 → "Structural Fabrication & Plant Erection Contractor, Chennai — Madras Swastic Engineers". Title → "Structural Fabrication Contractor Chennai | ISO 9001 | Madras Swastic Engineers".

HIGH 7No meta description, no canonical, no Open Graph tags

Social shares produce generic previews. Duplicate-URL signals between the Vercel slug and the custom domain are uncontrolled. Twitter/X cards do not render.

Fix: Add unique 150–160 char meta description + canonical + full OG/Twitter card metadata on every page.

HIGH 8No dedicated service / capability pages

Each capability (Structural Fabrication, Mechanical Erection, Piping & Skid, Turnkey) maps to a distinct high-intent procurement query. Competing vendor sites run 2,500–4,000 words across capability and portfolio sections.

Fix: Build four service pages, 800–1,200 words each, with material grades (IS 2062, ASTM A36), plate thickness range, weld procedure specs (GTAW/SMAW), NDT capabilities, equipment list (crane tonnage, transport reach), and one named project reference per page.

HIGH 9Marquee client claims are asserted, not evidenced

ISRO, Schwing Stetter, and Manali Petrochemicals logos appear with no project case studies, no scope (tonnage, timeline, location), no client-attributed quote. Logos alone score poorly with both Google E-E-A-T evaluators and AI citation engines.

Fix: One 300-word case study per anchor client: "ISRO LPSC, Mahendragiri — 340 MT structural package, delivered Q3 2023." Include client logo with descriptive alt text.

HIGH 10ISO 9001:2015 stated as text but never linked

No certificate number, no expiry date, no PDF download, no registrar verification link. Multinational procurement compliance teams (the kind that vet ISRO vendors) cannot proceed past this gap.

Fix: Publish the certificate PDF; add hasCredential with the registration number in schema; link from a homepage credibility bar.

HIGH 11No named principal / director bio

B2B industrial vendors need accountable humans. The email md@madrasswastic.com exists but no MD name, no engineering qualifications, no GST number, no CIN. Enterprise procurement teams running structured vendor qualification cannot complete this step.

Fix: About page with named MD, engineering credentials (AMIE / B.E. Mech / M.Tech), GST number, CIN (from MCA), and ISO certificate registration number.

HIGH 12No llms.txt

HIGH 13AI citability is structural, not editorial

Stats (96.4% on-time, 99% safety, 42,000 sq.ft., 27 years, ISRO client) all exist on the homepage, but they live inside isolated grid cards — not extractable as contiguous passages. Perplexity and ChatGPT extract contiguous 134–167-word passages; a number alone in a <div class="card"> is not citable.

Fix: One 90–130 word prose paragraph above the fold co-locating brand name + location + ISO cert + founding year + top three clients + one anchor stat. Highest-leverage single writing task for AI citation.

  • Live HTTP fetch was not available in this session — Bash and WebFetch tools were not permitted. Five of nine sub-agents drew their findings from the prior audit baseline. Re-run with network permission for fresh measurements.

  • Inner-page analysis is worst-case because every inner page returns 404. The four service pages, projects page, and contact page must be re-audited once they exist.
